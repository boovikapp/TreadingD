# IBKR → דוח מס ישראלי (₪)

## מבנה הפרויקט
```
ibkr-tax-il/
├── api/
│   └── process-report.js   ← ה-Serverless Function היחיד
├── lib/
│   ├── csvParser.js         ← פענוח קובץ ה-CSV הרב-סקציוני של IBKR
│   ├── boiRates.js          ← שליפת שערי בנק ישראל + caching + fallback
│   ├── taxCalculator.js     ← המרה לשקלים + חישוב רווח/הפסד
│   └── excelBuilder.js      ← בניית קובץ ה-Excel (2 לשוניות)
├── public/
│   └── index.html           ← הדף שאתה פותח בדפדפן
├── package.json
└── vercel.json
```

## התקנה והעלאה ל-Vercel
1. `npm install` בתיקיית הפרויקט (מתקין את `csv-parse` ו-`xlsx`).
2. `vercel` (או חבר את התיקייה ל-GitHub וייבא ב-vercel.com) → Deploy.
   Vercel מזהה אוטומטית שקבצים בתיקיית `api/` הם Serverless Functions,
   ושתיקיית `public/` היא הסטטי - אין צורך בקונפיגורציה נוספת.
3. פתח את ה-URL שקיבלת, גרור קובץ CSV ולחץ על הכפתור.

## איך להוציא CSV מ-IBKR
ב-IBKR: **Reports → Activity Statement → Custom Statement**, טווח תאריכים
= שנת המס כולה, פורמט = CSV. **חשוב:** בהגדרות ה-Custom Statement, תחת
Trades, הפעל את האופציה **"Closed Lot"** (Include closed lots) - זה מה
שנותן למערכת את תאריך הקנייה המדויק של כל פוזיציה שנסגרה, ולא רק את תאריך
המכירה. בלי זה, המערכת עדיין תעבוד אבל תסמן שורות עם "תאריך קנייה משוער"
במקום התאריך האמיתי (ראה הערה בקוד `csvParser.js`).

## הערה חשובה על ה-API של בנק ישראל
כתובת ה-API שסיפקת נבדקה - אך מבנה ה-JSON המדויק שהיא מחזירה לא היה ניתן
לאימות מלא מולי (אין לי גישת רשת חיה לבדוק תגובה אמיתית). לכן `boiRates.js`
כתוב כך שהוא מנסה כמה צורות תגובה נפוצות באופן אוטומטי (`data`, `Data`,
`observations` וכו'), ואם אף אחת לא מתאימה - הוא **זורק שגיאה ברורה
ומדפיס ל-logs את התגובה הגולמית** במקום לייצר שקט מספרים שגויים. בפעם
הראשונה שתריץ את זה בפרודקשן: אם תקבל שגיאה, פתח את ה-Logs בדשבורד של
Vercel, תראה שם את מבנה ה-JSON האמיתי, ותעדכן בהתאם רק את הפונקציה
`parseBoiResponse()` (השורות שמנסות `item.date` / `item.value` וכדומה) -
שאר המערכת (fallback, caching, חישובי מס) לא תלויה בזה ולא צריכה להשתנות.

## לוגיקת ה-Fallback לשערים
המערכת שולפת בקריאת API **אחת** את כל טווח התאריכים הדרוש (מה-10 ימים
שלפני העסקה המוקדמת ביותר ועד העסקה המאוחרת ביותר), בונה מפה של תאריך→שער
בזיכרון, ולכל עסקה בודקת: יש שער בדיוק לתאריך? אם לא - הולכת יום אחורה,
עד 10 ניסיונות. זה גם מממש caching (בקשת רשת אחת לכל הדוח, לא אחת לכל
עסקה) וגם fallback לסופ"ש/חג.

## מגבלות שכדאי לדעת
- החישוב מניח "מלאי עסקי" (מדרגות מס רגילות) - הלשונית "ריכוז להקלדה"
  נותנת לך את הרווח/הפסד הנקי בש"ח; שילוב שלו עם שאר ההכנסות שלך לפי
  המדרגות עצמן נעשה מחוץ למערכת (בדוח השנתי / עם רו"ח).
- דיבידנדים, ריבית, ומיסי withholding אינם כלולים - זו מערכת לחישוב רווח
  הון ממסחר (Trades) בלבד, בדיוק כפי שביקשת.
- מומלץ להריץ פעם ראשונה על קובץ קטן/לדוגמה ולבדוק ידנית 2-3 שורות מול
  ה-CSV המקורי לפני הגשה רשמית.
